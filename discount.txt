Окей, Михаил, давай сделаем нормальную нелинейную шкалу, где:

мелкие оплаты почти без скидки,

чем больше дней — тем быстрее растёт скидка,

но максимум остаётся 30%.

Самый логичный вариант — степенная кривая (power curve).
Она хорошо ложится и даёт "плавное начало → резкий рост к концу".

Формула

Берём степень p (обычно 1.5–3).
Чем больше p, тем медленнее растёт скидка в начале и сильнее в конце.

discountPercent = ( (days-1) / 364 )^p * 30


— при p=2 — мягкая кривая
— при p=3 — ещё более медленно в начале, стремительно в конце

Я поставлю p = 2.5 — хорошая компромиссная «красота».
Можешь менять по вкусу.

Функция TypeScript
export function calcPrice(basePrice: number, days: number, p: number = 2.5) {
    if (days < 1) days = 1;

    const ratio = (days - 1) / 364;           // 0..1
    const discountPercent = Math.min(
        Math.pow(ratio, p) * 30,
        30
    );

    const priceWithDiscount = basePrice * days * (1 - discountPercent / 100);

    return {
        price: Number(priceWithDiscount.toFixed(2)),
        discountPercent: Number(discountPercent.toFixed(2)),
    };
}
